# クイックスタートガイド

## 🚀 5分で始める出欠管理システム

このガイドでは、最短でアプリケーションを起動する方法を説明します。

---

## 前提条件

- Node.js 16.x以上がインストールされている
- PHP 7.4以上がインストールされている
- MySQL 5.7以上がインストールされている

---

## セットアップ手順

### 1. 依存関係のインストール（1分）

```bash
npm install
```

### 2. データベースの作成（1分）

```bash
mysql -u root -p < database_complete.sql
```

### 3. 設定ファイルの確認（1分）

`backend-php/config/config.php`を開いて、データベース接続情報を確認：

```php
define('DB_HOST', 'localhost');
define('DB_NAME', 'sotsuken');
define('DB_USER', 'server');
define('DB_PASS', 'pass');
```

必要に応じて変更してください。

### 4. 開発サーバーの起動（1分）

```bash
npm start
```

ブラウザで`http://localhost:3000`にアクセスします。

## アプリケーションの使い方

### 1. ログイン
- ブラウザで`http://localhost:3000`にアクセス
- ログインフォームにメールアドレスとパスワードを入力
- 「ログイン」ボタンをクリック

### 2. ダッシュボード
- ログイン後、ダッシュボードが表示されます
- 出欠記録の一覧を確認できます

### 3. カレンダー
- サイドバーから「カレンダー」をクリック
- 月別の出欠記録をカレンダー形式で確認できます

### 4. 学生管理
- サイドバーから「学生管理」をクリック
- 学生の一覧を確認できます
- 新規学生の追加が可能です

### 5. 出欠記録
- サイドバーから「出欠記録」をクリック
- 学生の出欠記録を確認・追加できます

## よくある質問

### Q: ログインできない
**A**: 以下を確認してください：
- データベースが正しく作成されているか
- `backend-php/config/config.php`のデータベース接続情報が正しいか
- MySQLが起動しているか

### Q: APIエラーが発生する
**A**: 以下を確認してください：
- フロントエンドのAPI URLが正しいか（`src/api/attendanceApi.js`）
- バックエンドのPHPファイルが正しく配置されているか
- PHPのエラーログを確認

### Q: CORSエラーが発生する
**A**: 以下を確認してください：
- `backend-php/config/config.php`のCORS設定
- フロントエンドのオリジンが正しく設定されているか

## トラブルシューティング

### データベース接続エラー

```bash
# MySQLの状態を確認
sudo systemctl status mysql

# MySQLを起動
sudo systemctl start mysql
```

### PHPエラー

```bash
# PHPのバージョンを確認
php -v

# 必要な拡張がインストールされているか確認
php -m | grep -E 'pdo|json|mbstring'
```

### Node.jsエラー

```bash
# Node.jsのバージョンを確認
node -v

# npmのキャッシュをクリア
npm cache clean --force

# node_modulesを再インストール
rm -rf node_modules package-lock.json
npm install
```
